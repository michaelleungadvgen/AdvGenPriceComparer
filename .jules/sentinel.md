## 2025-02-28 - Plaintext API Key Storage in Settings File
**Vulnerability:** API keys were being written to local JSON files (`settings.json`) in plaintext by the `SettingsService`, making them accessible to any user or application with file system read access.
**Learning:** Local application settings stored in AppData are often treated as "secure enough" by developers, but they remain highly vulnerable to local credential theft if unencrypted.
**Prevention:** Always encrypt sensitive settings (like API keys, passwords, or tokens) at rest. For Windows desktop applications, utilize `System.Security.Cryptography.ProtectedData` (DPAPI) bound to the `CurrentUser` scope, which seamlessly encrypts data using the user's OS credentials.
## 2023-11-20 - Ensure downloaded updates are cryptographically verified
**Vulnerability:** The application downloaded and automatically executed executable updates (.msi, .exe) via auto-update functionality (`UpdateService`) using `Process.Start()` without cryptographically verifying the integrity or authenticity of the downloaded payload.
**Learning:** Any auto-update mechanism that downloads executables over the network and runs them must ensure the payload is intact and from a trusted source. Failure to check file hashes allows for potential man-in-the-middle attacks or compromised CDNs to supply malicious executables. Empty or missing file hashes must be rejected.
**Prevention:** Always require and validate cryptographic hashes (e.g., SHA-256) of downloaded update files against a trusted manifest before writing the payload to disk or executing it. Fail closed if the hash is missing or mismatched.
