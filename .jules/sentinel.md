## 2024-05-24 - Cryptographic Verification for Downloaded Updates
**Vulnerability:** The application downloaded and executed an installer via Process.Start without verifying its integrity or authenticity, leading to a critical Remote Code Execution (RCE) vulnerability if the download was intercepted or tampered with.
**Learning:** Even when using HTTPS, relying solely on the download URL is insecure. The update mechanism must cryptographically verify downloaded installers using SHA-256 against a hash provided in the update manifest (UpdateCheckResult) before execution. This validation must also fail securely: if the hash is missing or empty, the update must be rejected to prevent bypasses via empty manifests.
**Prevention:** Always include a cryptographic hash in update manifests and explicitly verify the downloaded payload against this hash before executing it. Ensure that empty or missing hashes in the manifest result in a secure rejection rather than bypassing validation. When validating hashes in C#, use `Convert.ToHexString()` combined with an `OrdinalIgnoreCase` string comparison to ensure broader .NET version compatibility.
