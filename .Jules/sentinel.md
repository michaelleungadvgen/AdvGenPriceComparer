## 2025-03-10 - Unauthenticated Access to API Endpoints in Production
**Vulnerability:** The `/api/prices` endpoints were accessible without an API key because the middleware bypassed the check conditionally for all GET requests with that path.
**Learning:** Hardcoded path logic that skips authentication without explicitly checking the `IWebHostEnvironment.IsDevelopment()` can expose sensitive or administrative data in production. In ASP.NET Core convention-based middleware (e.g., `ApiKeyMiddleware`), Singleton services like `IWebHostEnvironment` can be safely injected via the constructor, while Scoped services (like database contexts or specific authentication services) must be injected into the `InvokeAsync` method to avoid lifecycle mismatch errors.
**Prevention:** Ensure that environment-specific bypasses for authentication explicitly check the hosting environment using `IsDevelopment()`. Consider making the API Key an explicit requirement across all endpoints and removing hardcoded path bypasses altogether.
